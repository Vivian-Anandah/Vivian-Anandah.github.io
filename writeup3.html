Metadata Capture via Social Engineering

Reconnaissance | CTF Write-up

← Back to Portfolio

Objective

In this challenge, my goal was to capture metadata from a target, including their IP address, approximate location, and device information. This was achieved by setting up a listener and delivering a crafted link to the target.

Tools Used

  • Seeker
  • Facad1ng
  • Serveo
  • Firefox browser

Step 1: Setting Up the Listener

I started by using Seeker to capture metadata when a user interacts with a link. After downloading it from GitHub, I launched the tool and selected a template.

I chose a Zoom-themed template to make the link appear legitimate and increase interaction.

http://localhost:8080

Accessing the local server confirmed the interface was working as expected.

Step 2: Exposing the Service

Since the service was only available on my machine, I needed to make it accessible externally. For this demo, I used Serveo to create a temporary public endpoint.

Normally, tools like Cloudflare Tunnel or ngrok are preferred because they are more stable and provide better monitoring capabilities, but Serveo is quick and requires minimal setup.

First, I prepared the lab environment.

Next, I created a tunnel using:

ssh -R 80:localhost:8080 serveo.net

This generated a public URL that forwarded requests directly to my local server.

https://10c7b342e1624a45-62-8-73-21.serveousercontent.com

As shown above, the generated link is clearly suspicious and easy to identify, which is where the next step becomes important.

Step 3: Link Obfuscation

Facad1ng is a social-engineering tool used for URL masking and redirection. It helps make suspicious-looking URLs appear more legitimate, such as the Serveo-generated link.

On another terminal, I navigated into the Facad1ng directory using:

cd Facad1ng

I then launched the tool with:

python3 facad1ng.py

Once started, I was prompted to enter the original URL, where I supplied the link generated by Serveo.

Next, I was asked to customize the domain name. I crafted it to resemble a legitimate Zoom invitation link. Since I did not require phishing keywords for this demonstration, I skipped that option and continued.

Facad1ng then generated two masked links. I selected the HTTPS version:

https://us01web1.zoom.us-@clck.ru/3TTSDQ

This made the link appear significantly more legitimate before sending it to the target.

Result

With the masked link finally generated, I was able to deliver it to the target.

Since this demonstration used the free version of Serveo, the target first encounters a warning page before proceeding, as shown below. Premium tunneling services generally provide a cleaner and more convincing experience without these warnings.

After clicking continue, the target is redirected to the Zoom-themed interface prepared earlier.

Once the target interacted with the page, Seeker successfully captured metadata from the session.

  • IP address
  • Approximate location
  • Device and browser details

As observed in the Seeker terminal, the initial output reveals the target’s IP address, device details, and approximate location—even when location permissions are denied or disabled. The subsequent output provides more detailed information, including a generated link that simplifies verifying the target’s location.

This demonstrates that meaningful metadata can still be collected despite restricted location permissions.

Conclusion

This demonstrates how easily metadata can be exposed through simple user interaction. It highlights the importance of awareness when handling links and external content.

Defensive Takeaways

  • Avoid clicking unknown or unverified links
  • Always verify sources
  • Use VPNs or privacy tools
  • Be cautious of links mimicking trusted services
